# Code of Conduct

## Our standard

This project aims to stay useful, technically honest, and respectful.

Examples of behavior that help:

- being respectful in disagreement
- assuming good faith while still being clear and direct
- giving actionable technical feedback
- reporting bugs and security concerns responsibly
- keeping discussion grounded in evidence, reproduction steps, and user impact

Examples of unacceptable behavior:

- harassment, personal attacks, or intimidation
- insulting or demeaning language
- deliberately derailing technical discussion
- publishing private or security-sensitive details irresponsibly
- spamming, trolling, or bad-faith participation

## Scope

This applies to repository discussions, issues, pull requests, and other project spaces.

## Enforcement

Project maintainers may edit, remove, or moderate comments, issues, pull requests, and other contributions that violate this code of conduct.

## Reporting

- for ordinary conduct problems, open an issue only if doing so is appropriate and does not create additional harm
- if the report should stay private, contact the maintainer directly through a non-public channel you already have, or ask for a private contact path without posting sensitive details publicly
- for security-sensitive or privacy-sensitive concerns, follow `SECURITY.md` instead of opening a detailed public report first

## Practical note

This is a small project, so moderation will favor clarity, safety, and keeping the project usable.
